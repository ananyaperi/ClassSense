Autonomous Classroom Monitoring and Assistance Robot

ClassSense

ClassSense is an autonomous classroom-support robot designed to monitor noise levels, detect distraction hotspots, navigate safely with obstacle avoidance, and visualize noisy and quiet zones through a web-based heatmap. The robot provides non-intrusive behavioral feedback using an onboard OLED display and motion toward noisy regions.

Problem Statement

Why this project matters

In classroom environments, noise can increase when teachers are focused on instruction and cannot continuously monitor every region of the room. This reduces attention, increases distractions, and adds extra workload to maintain discipline. A supportive autonomous system can monitor the environment, identify noisy regions, and encourage quiet behavior without interrupting the lesson.

Design Goals

What ClassSense targets
  • Real-time noise monitoring and hotspot identification
  • Safe indoor navigation with obstacle avoidance
  • Non-intrusive student feedback through motion and OLED messages
  • Web-based heatmap visualization and simple controls

Proposed Solution

What the robot does

ClassSense patrols within a defined 100 cm × 100 cm area while continuously reading sound intensity and obstacle distance. When noise crosses a defined threshold, the robot performs a direction-finding scan, rotates to face the loudest direction, and moves toward the hotspot. The corresponding grid cell is marked on a web heatmap and the robot displays a reminder message on the OLED screen.

Behavior: Patrol and respond Control: Hybrid (reactive + deliberate) Interaction: OLED messages

Working Methodology

End-to-end operation
  1. Initialization: ESP32 starts the WiFi hotspot, launches the web interface, and shows a default OLED reminder.
  2. Start command: The robot begins patrolling the defined area from the origin position.
  3. Patrol mode: Continuous sampling of sound intensity and obstacle distance during movement.
  4. Noise trigger: When sound crosses the threshold, the robot enters hotspot response mode.
  5. Direction finding: The robot performs a full rotation scan and selects the direction with maximum sound.
  6. Approach: The robot rotates to face the target direction and moves forward while monitoring safety constraints.
  7. Obstacle avoidance: If an obstacle is within the minimum distance, the robot stops and chooses a safer side.
  8. Boundary protection: The robot stays within the 100 cm × 100 cm boundary and ignores external noise sources.
  9. Heatmap marking: The current grid cell is marked as noisy and scoring is updated on the dashboard.
  10. Resume patrol: The robot returns to patrol immediately after marking the region.

Heatmap Model

4×4 grid scoring

The classroom area is discretized into a 4×4 grid. Each cell maintains a score. Positive scores represent quiet regions, while negative scores represent noisy regions. When a hotspot is detected, the corresponding cell is updated and highlighted on the web interface.

Hardware and Power

Build components
  • ESP32 DevKit microcontroller for sensing, control logic, WiFi hotspot, and web hosting
  • MAX9814 microphone module for analog sound intensity detection
  • HC-SR04 ultrasonic sensor for obstacle detection
  • TB6612FNG motor driver for dual motor control
  • Two DC gear motors for differential drive movement
  • 1.3 inch SH1106 OLED display for non-verbal reminders and teacher messages
Power separation: ESP32 powered via USB; motors powered via 4 AA batteries to reduce resets caused by motor noise.
Subsystem Power Source Reason
ESP32 + sensors + OLED USB power Stable supply for reliable sensing and WiFi hosting
Motors 4 AA batteries Isolates motor current spikes and electrical noise
