ClassSense

An Autonomous Classroom Monitoring and Assistance Robot. The robot patrols within a defined area, monitors noise levels, detects hotspots, avoids obstacles, and visualizes activity using a web-based 4×4 heatmap. It provides non-intrusive feedback using an OLED display.

Demo Interface Working Method Architecture

Quick Summary

Key points
  • Controller: ESP32 DevKit
  • Sensors: MAX9814 microphone, HC-SR04 ultrasonic
  • Drive: Differential drive using TB6612FNG and two DC motors
  • Display: 1.3 inch SH1106 OLED
  • Interface: WiFi hotspot with web dashboard and heatmap
  • Operating area: 100 cm × 100 cm, divided into 16 cells

Problem Statement

Why needed

During teaching, it is difficult to continuously monitor all regions of a classroom. Increased noise reduces student attention, creates distractions, and increases the teacher’s workload. A supportive autonomous system can monitor the classroom environment and encourage quieter behavior without interrupting instruction.

Objectives

Project goals
  • Monitor noise levels in real time
  • Detect and approach noisy regions
  • Avoid obstacles while moving
  • Stay inside a defined boundary
  • Update a grid-based heatmap
  • Show reminders on OLED

Proposed Solution

What it does

The robot patrols inside a 100 cm × 100 cm area while continuously reading sound intensity and obstacle distance. When the sound crosses a threshold, it performs a rotation scan to find the loudest direction, turns toward it, moves toward the hotspot, updates the heatmap cell, and displays a reminder message on the OLED.

The system uses a hybrid control approach: reactive behaviors for obstacle avoidance and noise response, and deliberate behaviors for patrol strategy, grid scoring, and boundary constraints.

System Architecture

Modules
Module Components Role
Sensor Input MAX9814, HC-SR04 Noise intensity and obstacle distance sensing
Processing ESP32 DevKit Decision making, WiFi hotspot, web server
Motion TB6612FNG, 2 DC motors Differential drive movement and rotation scanning
Feedback SH1106 OLED Reminder display and teacher messages
Dashboard ESP32 web UI Start/stop control and heatmap visualization

Working Methodology

Steps
  1. Initialize WiFi hotspot and OLED message
  2. Wait for start command from web dashboard
  3. Patrol and continuously sample sound and distance
  4. When noise threshold is reached, rotate and find maximum sound direction
  5. Move toward the hotspot while avoiding obstacles
  6. Update the current heatmap cell score and mark noisy regions
  7. Resume patrol
Operating area: 100 cm × 100 cm Grid: 4 × 4 (each cell 25 cm × 25 cm) Quiet condition: sound below 700 Noise condition: sound at or above 1200 Obstacle condition: distance below 20 cm
